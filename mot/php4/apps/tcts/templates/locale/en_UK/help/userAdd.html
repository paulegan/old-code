TCTS Help :: Overview :: Add Users


The number of rows in the permission matrix will change depending on how many centres you have the "manage users" privilege. If only one centre is allowed, the row title will be "Access" rather than the centre name.

Users with the "submit batch" privilege will appear in the "Submitted By" lists for the appropriate center.

1.

Enter a list of user IDs, 1 per line. The user ID must be a valid core directory user ID if the user is log on to the system. If you simply wish to add an entry to the "Submitted By" lists (by assigned the user the "submit batch" privilege) the user ID does not have to be valid. If you enter a commerce ID it will automatically be converted to the correct user ID.

2.

Click on the "Check User IDs" button to validate all user IDs entered. All valid user IDs will be followed by that user's name. You must remove any users that already exist. If the user ID was not found to be a valid employee the user's name will be set to "Name Unknown"; you may change this.

3.

Change the permissions as required by selecting/unselecting the checkboxes in the permission matrix. When a checkbox is selected this indicates that the user will be permitted to perform the operation specified by the column title for the center specified by the row title. Alternatively you can select a role from the "Role" list. These defined common permission combinations.

4.

Click the "Add Users" button to add all the user's listed to the system with the specified privileges.

5.

Click "Edit Users" to open the user update page. Any user's listed will not be added - first use the "Add Users" button.
